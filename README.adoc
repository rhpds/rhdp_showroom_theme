= RHDP Showroom Theme
:url-project: https://github.com/rhpds/rhdp_showroom_theme
:url-antora-docs: https://docs.antora.org
:url-nodejs: https://nodejs.org

Custom https://antora.org[Antora] UI bundle for the Red Hat Demo Platform Showroom.

== Use the Theme

Add the following to your Antora playbook (`site.yml`):

[source,yaml]
----
ui:
  bundle:
    url: https://github.com/rhpds/rhdp_showroom_theme/releases/download/latest/ui-bundle.zip
    snapshot: true
----

TIP: The `snapshot` flag tells Antora to re-fetch the bundle when `--fetch` is passed on the command line.
The `latest` release is updated automatically whenever the `latest` branch is pushed.

== Navbar Branding

Set `navbar_logo` in `site.keys` to choose a logo.
That single key controls everything:

* *Image* -- loaded from `src/img/logo-<stem>.svg` in the theme.
* *Link* -- always points to the site root so the user stays inside the iframe.
* *Alt text* -- looked up automatically from a map in `src/helpers/navbar-logo-alt.js`.

When no key is set the default *Red Hat Demo Platform* branding is used.

=== Available logos

[cols="1,3,2"]
|===
| Stem | SVG file | Alt text

| `demo-platform` (default)
| `src/img/logo-demo-platform.svg`
| Red Hat Demo Platform

| `summit`
| `src/img/logo-summit.svg`
| Red Hat Summit
|===

=== Set branding in site.yml

[source,yaml]
----
site:
  keys:
    navbar_logo: summit
----

=== Override at build time via CLI

[source,bash]
----
antora --key navbar_logo=summit \
       --ui-bundle-url=https://github.com/rhpds/rhdp_showroom_theme/releases/download/latest/ui-bundle.zip \
       site.yml
----

=== How it works

The Handlebars template `src/partials/header-content.hbs` renders the navbar logo like this:

[source,html]
----
<a href="{{siteRootPath}}">
  <img src="{{uiRootPath}}/img/logo-{{or site.keys.navbarLogo 'demo-platform'}}.svg"
       alt="{{navbar-logo-alt (or site.keys.navbarLogo 'demo-platform')}}">
</a>
----

`navbar-logo-alt` is a custom Handlebars helper (`src/helpers/navbar-logo-alt.js`) that maps the logo stem to its alt text.

=== Adding a new logo

. Add the SVG as `src/img/logo-<stem>.svg` (e.g. `src/img/logo-connect.svg`).
. Open `src/helpers/navbar-logo-alt.js` and add the stem and alt text to the map:
+
[source,js]
----
const ALT_TEXT = {
  'demo-platform': 'Red Hat Demo Platform',
  summit: 'Red Hat Summit',
  connect: 'Red Hat Connect',   // <-- new entry
}
----
. Rebuild the theme (`npx gulp bundle`) and release.

== Other Customizations

=== External links dropdown

Add links to the navbar dropdown via Antora page attributes:

.content/antora.yml
[source,yaml]
----
asciidoc:
  attributes:
    page-links:
    - url: https://redhat.com
      text: Red Hat
    - url: https://www.redhat.com/summit
      text: Summit
----

=== User display

Show the current user's name in the navbar:

.content/antora.yml
[source,yaml]
----
asciidoc:
  attributes:
    user: 'User X'
----

== Releases

A GitHub Actions workflow builds and publishes `ui-bundle.zip` automatically.
Two release channels are supported:

=== Rolling `latest` release

The `latest` release is updated on every push to the `latest` branch.
Most workshops should point here -- they always get the newest theme.

To ship an update:

. Merge your changes to `main`.
. Fast-forward the `latest` branch:
+
 $ git checkout latest && git merge --ff-only main && git push

CI replaces the `latest` release asset automatically.

=== Tagged releases

Push a semver tag to create a pinned release for testing or rollback:

 $ git tag v2.1.0 && git push origin v2.1.0

CI creates a `v2.1.0` release with the built bundle.
Use it to test before promoting to `latest`:

[source,yaml]
----
ui:
  bundle:
    url: https://github.com/rhpds/rhdp_showroom_theme/releases/download/v2.1.0/ui-bundle.zip
----

== Development

=== Prerequisites

* {url-nodejs}[Node.js] >= 20

=== Install dependencies

 $ npm ci

=== Preview the theme

 $ npx gulp preview

Opens a live-reloading preview server on `http://localhost:5252`.

=== Build the UI bundle

 $ npx gulp bundle

Produces `build/ui-bundle.zip`, which you can reference locally:

 $ antora --ui-bundle-url=build/ui-bundle.zip site.yml
