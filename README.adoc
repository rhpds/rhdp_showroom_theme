= RHDP Showroom Theme
:url-project: https://github.com/rhpds/rhdp_showroom_theme
:url-antora-docs: https://docs.antora.org
:url-nodejs: https://nodejs.org

Custom https://antora.org[Antora] UI bundle for the Red Hat Demo Platform Showroom.

== Use the Theme

Add the following to your Antora playbook (`site.yml`):

[source,yaml]
----
ui:
  bundle:
    url: https://github.com/rhpds/rhdp_showroom_theme/releases/latest/download/ui-bundle.zip
    snapshot: true
----

TIP: The `snapshot` flag tells Antora to re-fetch the bundle when `--fetch` is passed on the command line.

== Navbar Branding

The navbar logo, link, and alt text are configurable via `site.keys`.
When no keys are set the default *Red Hat Demo Platform* branding is used.

[cols="1,2,1"]
|===
| Key | Description | Default

| `navbar_logo`
| Filename stem of the logo SVG in `img/` (e.g. `summit`, `rh1`)
| `demo-platform`

| `navbar_logo_url`
| URL the logo links to
| `\https://demo.redhat.com`

| `navbar_logo_alt`
| Accessible alt text for the logo image
| `Red Hat Demo Platform`
|===

Available logos shipped with the theme: `demo-platform`, `summit`.

=== Set branding in site.yml

[source,yaml]
----
site:
  keys:
    navbar_logo: summit
    navbar_logo_url: https://www.redhat.com/summit
    navbar_logo_alt: Red Hat Summit
----

=== Override at build time via CLI

The `--key` option has the highest precedence and overrides anything in `site.yml`.
This is useful when a build system controls branding across many workshops:

[source,bash]
----
antora --key navbar_logo=summit \
       --key navbar_logo_url=https://www.redhat.com/summit \
       --key navbar_logo_alt='Red Hat Summit' \
       --ui-bundle-url=https://github.com/rhpds/rhdp_showroom_theme/releases/latest/download/ui-bundle.zip \
       site.yml
----

== Other Customizations

=== External links dropdown

Add links to the navbar dropdown via Antora page attributes:

.content/antora.yml
[source,yaml]
----
asciidoc:
  attributes:
    page-links:
    - url: https://redhat.com
      text: Red Hat
    - url: https://www.redhat.com/summit
      text: Summit
----

=== User display

Show the current user's name in the navbar:

.content/antora.yml
[source,yaml]
----
asciidoc:
  attributes:
    user: 'User X'
----

== Development

=== Prerequisites

* {url-nodejs}[Node.js] >= 20

=== Install dependencies

 $ npm ci

=== Preview the theme

 $ npx gulp preview

Opens a live-reloading preview server on `http://localhost:5252`.

=== Build the UI bundle

 $ npx gulp bundle

Produces `build/ui-bundle.zip`, which you can reference locally:

 $ antora --ui-bundle-url=build/ui-bundle.zip site.yml
